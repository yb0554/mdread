# mdread 实施路线与完成状态

## v1.2.3 交互体验补丁

- 顶部工具栏将“添加工作区”和“收藏”提升为直接可见操作，低频操作统一收纳到精简的“更多”菜单。
- 最近打开列表使用简洁文件名显示；完整路径仅在悬停与右键/键盘上下文菜单中提供，并会去除 Windows `\\?\` 前缀。
- Windows GitHub Release 自动附加便携版可执行文件，并把它纳入 SHA-256 校验清单。

本文将最初的 12 周升级计划映射为 v1.2.2 的可交付工程状态。优先级始终是安全、稳定性和可验证性。

## 已完成：v1.2.2 核心能力

- **单一授权边界**：`WorkspaceRegistry`、文档授权 ID、canonicalize 校验、相对路径限制、工作区移除后的授权/缓存/监听清理。
- **稳定读取生命周期**：结构化 `AppError`、编码元数据、25 MiB 二次确认、单活动监听器、220 ms 去抖、请求 ID latest-wins。
- **统一渲染与性能保护**：文档首次加载、重载、图片解析、DOMPurify、代码复制、大纲和进度条使用同一流水线；Markdown 解析与高亮转入 Worker，并支持延迟高亮/延迟大纲。
- **工作区体验**：目录失败重试、刷新、树状态不被搜索破坏、未展开目录搜索、`content:` 按需正文搜索、最近文件、收藏、会话恢复、侧边栏宽度恢复。
- **安全的外部行为**：默认阻止远程图片，外链白名单，系统默认应用打开，危险协议拒绝。
- **系统集成**：`.md` / `.markdown` 文件关联、首次启动参数读取、single-instance 第二实例文件转交，以及“在文件管理器中显示当前文档”。
- **无障碍基础**：语义化按钮、标签、焦点环、文件树与大纲方向键导航。
- **测试与发布工程**：Vitest / jsdom、Rust 单测、Windows 嵌入式 WebdriverIO 桌面冒烟 E2E、覆盖率阈值、三平台 CI、RustSec 与 npm 审计、版本一致性检查、CycloneDX SBOM 和递归 SHA-256 清单。


## v1.2.2 稳定性补丁

- **可撤销单文件授权**：独立文档在切换、打开失败或被替换时释放授权、资源 scope 与监听；移除父工作区时会重建仍有效的嵌套工作区 scope。
- **竞态和搜索取消**：远程图片偏好不会取消正在加载的新文档；工作区搜索带请求 ID、取消标记和界面状态，旧扫描不得更新当前结果。
- **真实 UI 验证**：文件树统一实际 DOM/CSS 层级，长中英文文件名及 `__pycache__` 类目录按省略号截断；右侧目录可滚动，在小于 960px 的窗口中作为抽屉打开。
- **格式与无障碍一致性**：选择器、拖放、系统打开和 Rust 后端仅接受 `.md` / `.markdown`；最近文件、目录树和清理操作使用语义按钮并支持键盘导航。
- **发布透明度**：v1.2.2 Windows 包未签名；正式 Release 自动附加 SBOM 和 SHA-256 清单，不将未验证的签名、离线 Runtime 或自动更新宣称为已启用。

## 发布前仍需外部条件的事项

以下事项不由源代码本身决定，必须在拥有真实发布凭据、发布基础设施或干净系统时执行；它们不会在本地构建中伪装为“已完成”。

1. **代码签名与公证**：Windows 证书签名、macOS Developer ID 签名和 notarization 需要对应证书、密钥与 CI secrets。
2. **自动更新通道**：Tauri updater 需要稳定的 HTTPS 更新清单、签名私钥和公钥配置；在这些条件具备前保持未启用状态。
3. **Windows 离线 WebView2 发行包**：默认 NSIS bootstrapper 与便携主程序可构建；完全离线 Runtime 应以独立下载/安装包方式经过洁净 Windows 验证后再发布。
4. **跨平台安装矩阵**：Windows 嵌入式 E2E 已纳入 CI；Linux、macOS 的安装、升级、卸载与系统 WebView 依赖仍须在对应洁净系统持续验证。

## 验收清单

- 快速连续打开 A/B 文档，旧文档不能覆盖新文档。
- 连续打开 100 个文件，只有一个有效监听器。
- 搜索清空后原有目录折叠状态不变；未展开目录文件可搜索。
- 工作区移除后读取、图片资产和监听均被拒绝。
- 路径穿越、符号链接越界、`javascript:`/`file:` 链接、默认远程图片均被拒绝。
- UTF-8、UTF-8 BOM、GBK、解码警告及 1/5/25 MiB 文件按预期处理。
- 键盘可完成添加工作区、搜索、打开文档、切换目录和打印。
- Windows bootstrapper 与离线安装路径分别在洁净系统完成验证后才作为正式发布结论。
